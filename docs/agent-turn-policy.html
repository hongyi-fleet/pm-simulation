Agent Turn Policy

When does the PM agent act? And why that's a design decision, not an accident.

The PM agent is an AI. It has no attention bottleneck. When someone sends it a message, it responds immediately. The 30-minute interval is a proactive patrol, not a response delay.

This is a deliberate design choice: we simulate the constraints that actually matter for AI agents (information discovery, prioritization, action budget) and skip the ones that don't (notification delays, attention switching).

Why Not Simulate Notification Delays?

Human PM Bottlenecks
  • In a meeting, can't check Slack
  • Focused on writing, misses notification
  • Didn't hear the ping
  • Checking email but not chat
  • Attention split across 6 tools
AI Agent Bottlenecks
  • Information scattered across 6 tools
  • NPCs don't volunteer information
  • Must connect signals across tools
  • 5 write actions per turn budget
  • Time pressure (Friday deadline)

Simulating human attention delays for an AI agent tests the wrong thing. It penalizes the agent for a constraint it doesn't have, while missing the constraints that actually matter: information discovery, prioritization, and action under pressure.
